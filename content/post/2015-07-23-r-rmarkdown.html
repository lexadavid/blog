You might know it from any engineering job, you look for quick solution of your problem but the only available solution was created in other progrming language. While many engineers need to create solution themselves data related jobs do not usually have much limitation regarding to what technology they can use. So you often adopt strategy of using whatever makes the job. Most of data scientist then limit their choices for R and Python. Some tasks are simpler to do in Python and some tasks have just better package support in R. Yes, to know multiple technologies can save you a lot of time. But what if there is task where easiest solution combines multiple languages in the same time? One solution is to wrap one chunk of code to other language by using for example rpy2. That works well as long as you handle a bit challenging installation. Other solution is using Apache Airflow, platform for scheduling and monitoring workflows originally developed by Airbnb that enables to run each task as bash command and therefore is practically language independent.

What is Airflow?

Shortly it is data flow management tool to handle single time jobs (that you donâ€™t want to run on your local computer) as well as huge ETL processes in scale. Among the benefits you would find: - GUI that provides nice visualization for job process - Couple of metrics about your jobs - Easy access to logs when something goes wrong detail. - Scalability - Well documented - Active community - And many more
